# API & Signaling Protocol Specification

This document provides a detailed specification of the REST APIs and the WebSocket-based signaling protocol used in **FaceMe** to coordinate the peer-to-peer WebRTC connection.

---

## 1. REST API Endpoints

The Go backend exposes a few HTTP endpoints to query and initialize rooms. By default, these run on port `8080`.

### 1.1 Create Room

Initialize a room before peers connect.

* **Endpoint:** `POST /api/rooms`
* **Content-Type:** `application/json`
* **Request Body:**

  ```json
  {
    "id": "room-unique-code",
    "topic": "Project Demo Discussion"
  }
  ```

* **Success Response (200 OK):**

  ```json
  {
    "id": "room-unique-code",
    "topic": "Project Demo Discussion"
  }
  ```

* **Error Responses:**
  * `400 Bad Request` (Invalid JSON or empty room ID)

    ```json
    { "error": "Room ID is required" }
    ```

### 1.2 Get Room Details

Retrieve the status/metadata of a specific room.

* **Endpoint:** `GET /api/rooms/{room_id}`
* **Success Response (200 OK):**

  ```json
  {
    "id": "room-unique-code",
    "topic": "Project Demo Discussion"
  }
  ```

* **Error Responses:**
  * `404 Not Found` (Room does not exist)

    ```json
    { "error": "Room not found" }
    ```

---

## 2. WebSocket Signaling Endpoint

The real-time handshake process is conducted over WebSockets.

* **Endpoint:** `WS /ws/{room_id}`
* **Connection Handshake:**
  * When a peer connects, the server upgrades the HTTP request to a WebSocket connection.
  * The server maps the client to the specified `{room_id}`.
  * **2-Occupant Guard:** If the room already has 2 active connections, any 3rd client attempting to connect is immediately sent an error message and disconnected:

    ```json
    { "error": "Room is full" }
    ```

---

## 3. Signaling Message Protocol

The signaling server operates as a transparent pub/sub router. All messages sent by a client are forwarded directly to the *opposite* peer in the same room. Messages must follow a strict JSON payload structure:

```typescript
type SignalingMessage = {
  type: 'knock' | 'admit' | 'reject' | 'ready' | 'offer' | 'answer' | 'ice_candidate' | 'action';
  payload?: any;
};
```

### 3.1 Message Types & Payloads

#### `knock`

* **Direction:** Guest Client $\rightarrow$ Server $\rightarrow$ Room Creator
* **Description:** Sent by the guest client immediately upon WebSocket connection to request entry into the call.
* **Payload:**

  ```json
  {
    "type": "knock",
    "payload": {
      "name": "Alice"
    }
  }
  ```

#### `admit`

* **Direction:** Room Creator $\rightarrow$ Server $\rightarrow$ Guest Client
* **Description:** Sent by the room creator to approve the guest's knock request.
* **Payload:**

  ```json
  {
    "type": "admit",
    "payload": {
      "name": "Bob (Creator)"
    }
  }
  ```

#### `reject`

* **Direction:** Room Creator $\rightarrow$ Server $\rightarrow$ Guest Client
* **Description:** Sent by the room creator to deny entry to the knocking guest.
* **Payload:**

  ```json
  {
    "type": "reject"
  }
  ```

  *(Upon receipt, the guest's browser stops all local media tracks, closes the WebRTC peer connection, and terminates the WebSocket connection).*

#### `ready`

* **Direction:** Guest Client $\rightarrow$ Server $\rightarrow$ Room Creator
* **Description:** Sent by the guest after being admitted. Tells the creator that the guest is ready to receive a WebRTC SDP Offer.
* **Payload:**

  ```json
  {
    "type": "ready"
  }
  ```

#### `offer`

* **Direction:** Room Creator $\rightarrow$ Server $\rightarrow$ Guest Client
* **Description:** The WebRTC Session Description Protocol (SDP) offer containing local media codecs and configuration generated by the creator.
* **Payload:** Contains the browser's standard SDP object.

  ```json
  {
    "type": "offer",
    "payload": {
      "type": "offer",
      "sdp": "v=0\r\no=- 4210492810488279893 2 IN IP4 127.0.0.1\r\ns=-\r\nt=0 0\r\n..."
    }
  }
  ```

#### `answer`

* **Direction:** Guest Client $\rightarrow$ Server $\rightarrow$ Room Creator
* **Description:** The WebRTC SDP answer returning local codec/configuration from the guest.
* **Payload:** Contains the browser's standard SDP object.

  ```json
  {
    "type": "answer",
    "payload": {
      "type": "answer",
      "sdp": "v=0\r\no=- 5230491823904812304 2 IN IP4 127.0.0.1\r\ns=-\r\nt=0 0\r\n..."
    }
  }
  ```

#### `ice_candidate`

* **Direction:** Peer $\leftrightarrow$ Server $\leftrightarrow$ Peer (Bi-directional)
* **Description:** Exchanged asynchronously whenever a new Ice Candidate (network route path) is discovered by either browser's RTCPeerConnection engine.
* **Payload:** Standard `RTCIceCandidateInit` object.

  ```json
  {
    "type": "ice_candidate",
    "payload": {
      "candidate": "candidate:842163049 1 udp 1686052607 192.168.1.50 51234 typ host ...",
      "sdpMid": "0",
      "sdpMLineIndex": 0
    }
  }
  ```

#### `action`

* **Direction:** Peer $\leftrightarrow$ Server $\leftrightarrow$ Peer (Bi-directional)
* **Description:** Broadcasts transient in-call interactions to synchronize state overlays.
* **Payload:** Custom payload denoting the action (e.g. `raise_hand`).

  ```json
  {
    "type": "action",
    "payload": {
      "action": "raise_hand"
    }
  }
  ```
